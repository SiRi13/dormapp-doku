Use-Case: Fehlerbehandlung
Ziel: Ablauf einer Fehlerroutine
Kategorie: primär (erforderlich, häufig verwendet)
Vorbedingung: Ein Fehler muss aufgetreten sein
Auslösendes Ereignis: Der Fehler
Beschreibung:
1 Keine Verbindung zum Datenbankserver / Server antwortet nicht: Die bereits
zwischengespeicherten Daten lokal auf dem Handy aufbewahren und bei nächster
Datenbankverbindung automatisch mit auf den Server pushen. Dem Benutzer muss zu Anfang der
Fehlerroutine ein Hinweis gegeben werden.
2 Textfeld Fehler (Pflichtfeld nicht ausgefüllt, Format stimmt nicht): Benutzer den Hinweis
geben welcher Fehler vorliegt und dem Benutzer die Möglichkeit geben das Textfeld zu bearbeiten
3 Betriebssystem (Google Android) "killt" die Applikation: Den aktuellen Status lokal speichern.
Bei nächstem Start der App während des Startvorgangs den zuletzt lokal gespeicherten Status
laden. Dem Benutzer muss kein Hinweis gegeben werden.
